# 优化报告生成结果卡片

## 背景

报告导出完成后，聊天区原本显示偏系统日志的文本，例如“已调取 node3 数据生成 HTML”。这暴露了内部节点细节，也没有把“文件已生成、保存在哪里、能做什么”以清晰的产品化结果呈现给用户。

## 目标

- 把报告导出完成结果呈现为聊天内结果卡片。
- 隐藏 `node3` 等内部实现细节，改用“产品设计报告”“营销方案报告”等业务标签。
- 保留 AI 生成文件双归档规则：目标分类文件夹 + `AI 对话产出`，导出报告继续保存到 `编程文件库`。
- 历史会话重新打开时也能恢复报告卡片，而不是回退成原始日志文本。

## 范围

- 后端导出/报告生成事件输出更干净的 `content` 文案和结构化 `file_saved` 字段。
- 任务运行时在保存会话时写入隐藏的报告卡片 marker。
- 前端 reducer 将报告类 `file_saved` 事件转换为 `reportCards`。
- 聊天面板新增报告结果卡片组件和样式。
- 历史会话加载时解析隐藏 marker 并恢复卡片。

## 非目标

- 不改任务创建、SSE 解析、会话保存归属。
- 不改项目库文件 CRUD 或下载接口。
- 不改报告正文生成模板和节点路由规则。

## 验收

- 报告完成消息不再显示“调取 nodeX 数据”等内部日志。
- 卡片展示文件名、报告类型、生成时间、保存文件夹标签。
- 卡片提供预览和下载入口。
- `python -m compileall kaiwuback/server` 通过。
- `npm run build` 通过。
