# 完善项目库文件夹上传搜索

## Goal

补齐项目库的真实文件管理体验：用户可以创建项目库文件夹，将本地文件通过点击或拖拽上传到指定文件夹，在项目库里按名称搜索文件夹和文件，并能从最近文件进入文件夹详情页后返回项目库。

## Requirements

- 项目库支持新建文件夹，创建后立即显示在文件夹列表，并可作为上传目标。
- 上传弹窗支持点击选择本地文件和拖拽文件，上传到用户选择的项目库文件夹。
- 删除上传弹窗中被截图圈出的文件类型快捷按钮区域。
- 项目库最近文件中的文件夹/详情体验需要有返回按钮，避免进入详情后无法回到项目库首页。
- 搜索框按名称搜索，覆盖文件夹名和文件名；搜索结果随输入实时过滤。
- 保留现有 AI 生成文件的双归档规则，不改变 `AI 对话产出` 等固定中文文件夹名。

## Acceptance Criteria

- [ ] 点击“新建文件夹”填写名称并确认后，后端创建对应项目库目录，前端刷新并显示该文件夹。
- [ ] 上传文件弹窗点击上传区域能打开本地文件选择器，拖拽文件到上传区域也能选中文件。
- [ ] 选择文件夹并上传后，文件保存到 `kaiwuback/project-files/<文件夹>/`，项目库文件列表刷新可见。
- [ ] 上传弹窗不再显示 `JPG / MD / PDF / XLSX` 四个属性按钮。
- [ ] 项目库搜索框输入名称后，文件夹和文件列表按名称过滤，清空后恢复。
- [ ] 文件夹详情页有明确返回按钮，点击回到项目库首页/上一层。
- [ ] `python -m compileall kaiwuback/server` 通过。
- [ ] `npm run build` 在 `kaiwu/` 下通过。

## Definition of Done

- 前后端类型和接口保持一致。
- 错误反馈简洁，不泄露文件内容。
- 不触碰已有未关联的工作区改动。
- Trellis 检查完成，必要的新规范知识已评估是否需要记录。

## Technical Approach

- 后端在 `kaiwuback/server/api/routes_files.py` 追加项目库文件夹列表、创建文件夹、项目库文件上传接口。
- 前端在 `App.tsx` 持有项目文件夹、搜索词和刷新方法；`MainStage.tsx` 渲染搜索、过滤列表和文件夹详情页；`AppModals.tsx` 处理新建文件夹和文件上传表单。
- 文件夹列表将后端真实目录与现有静态文件夹元数据合并，避免丢失既有视觉描述。

## Decision (ADR-lite)

**Context**: 项目库已有静态文件夹配置和真实文件列表，但创建、上传、搜索还未接入真实数据。

**Decision**: 采用薄 API + 前端刷新模式，后端只负责目录/文件边界和基础校验，前端继续由 `App.tsx` 持有项目库服务端状态。

**Consequences**: 实现范围小，符合现有架构；后续若需要重命名/移动/删除，可在同一接口层扩展。

## Out of Scope

- 文件夹删除、重命名、权限管理。
- 文件内容全文搜索。
- 上传进度条、断点续传、大文件分片。
- 改动对话输入区的“上传文件”上下文功能。

## Technical Notes

- 现有项目库 UI 在 `kaiwu/src/features/layout/MainStage.tsx` 和 `kaiwu/src/features/layout/AppModals.tsx`。
- 现有项目库文件接口在 `kaiwuback/server/api/routes_files.py`，当前 `/api/project-files` 只列部分扩展名。
- 规范要求项目库文件夹中文名保持一致，尤其是 `AI 对话产出`。
