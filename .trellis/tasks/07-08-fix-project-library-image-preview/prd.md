# 修复项目库图片预览

## Goal

项目库详情页应能直接浏览常见图片文件，避免 JPG 等图片被展示为“此文件类型暂不支持内嵌预览”。这可以让用户在项目库中检查 AI 对话产出和手动上传的图片，而无需每次新窗口打开或下载。

## What I Already Know

* 用户截图显示项目库详情页正在查看 `AI生图_97245487_20260707105727.jpg`，文件类型标记为 `JPG`，但预览区显示不支持内嵌预览。
* 项目库文件详情页位于前端 `kaiwu/src/features/layout/MainStage.tsx`。
* 后端项目文件列表入口位于 `kaiwuback/server/api/routes_files.py`，文件 URL 由 `/project-files/{folder}/{filename}` 提供。

## Assumptions

* 修复应优先发生在项目库文件详情页，而不是单独的图片库弹层。
* 常见图片扩展名应该使用 `<img>` 内嵌预览；现有 HTML/PDF 等可继续使用当前 iframe 预览。
* “新窗口打开”和“下载文件”按钮保持现有行为。

## Requirements

* 项目库详情页识别常见图片文件类型并直接展示图片。
* 图片预览应使用项目文件的已有 URL，不新增重复下载链路。
* HTML/PDF/TXT/MD/CSV/JSON 继续沿用现有 iframe 内嵌预览。
* 非图片且不在现有可预览范围内的文件继续显示不支持内嵌预览。
* 预览区域在桌面尺寸下应保持稳定布局，图片完整展示且不拉伸变形。
* 所有项目库文件夹详情页都应提供与“返回项目库”同层级的编辑能力。
* 文件夹详情页应支持重命名当前文件夹、进入选择模式、删除所选条目。
* 图片库详情页删除所选图片时应真实删除后端原图、WebP 预览缓存、图片元数据和图片库归档副本。
* 普通文件夹详情页删除所选文件时应复用项目文件删除接口并只弹出一次确认。

## Acceptance Criteria

* [ ] 打开 `.jpg` / `.jpeg` 项目文件时，详情页预览区显示图片本体。
* [ ] 打开 `.png` / `.webp` / `.gif` 项目文件时，详情页预览区显示图片本体。
* [ ] 已支持的 HTML/PDF 等文件继续按原方式预览。
* [ ] 不支持的文件类型仍显示“此文件类型暂不支持内嵌预览”。
* [ ] 打开任意项目库文件夹详情页，可以在顶部点击“重命名”修改当前文件夹名称。
* [ ] 打开任意普通文件夹详情页，可以进入编辑模式并批量删除选中文件。
* [ ] 打开图片库详情页，可以进入编辑模式并批量删除选中图片，刷新后图片不再出现在图片库。
* [ ] 前端构建通过。

## Definition of Done

* 代码遵循前端组件、类型和样式规范。
* 运行前端构建验证。
* 若发现跨前后端契约问题，再运行后端语法检查。

## Out of Scope

* 不重做项目库详情页整体 UI。
* 不改变文件上传和下载流程。
* 不新增图片缩放、旋转、标注等编辑能力。

## Technical Notes

* 相关代码初步定位：`kaiwu/src/features/layout/MainStage.tsx`、`kaiwu/src/styles.css`。
* 后端 `_project_file_entry()` 返回的 `type` 来自 `f.suffix.upper().lstrip(".")`，所以 `.jpg` 会成为 `JPG`。
* 当前 `PROJECT_FILE_PREVIEW_TYPES` 仅包含 `HTML`、`HTM`、`PDF`、`TXT`、`MD`、`CSV`、`JSON`，导致 `JPG` 走不支持预览分支。
* 技术方案：新增图片类型集合；图片类型使用 `<img>` 渲染，文档类型继续使用 iframe。
* 文件夹 API 返回 `id` / `kind` 作为稳定身份；虚拟图片库、视频库可改显示名，但内部路径保持稳定。
* 图片批量删除使用 `DELETE /api/project-images`，请求体为 `{ "names": string[] }`。
