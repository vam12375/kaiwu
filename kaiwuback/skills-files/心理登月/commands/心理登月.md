# /心理登月

## 用法

```
/心理登月
```

或者带具体问题：

```
/心理登月 <你要分析的技术难点或产品困境>
```

## 这个命令做什么

1. 从对话上下文中提取最近的技术难点
2. 进行三层追问：表层抱怨 → 实际需求 → 情绪诉求
3. 从五个心理维度（体验重塑、时间感知、感官替代、社会证明、对比锚定）探索替代方案
4. 输出结构化分析报告：技术难点、深层需求、2-4个心理登月方案、成本对比、推荐行动

## 示例

```
用户：我们想提高App的打开速度，但后端优化需要重构整个数据层，预估3个月

/心理登月

→ 输出：深层需求不是"打开快"，而是"不要让我等的时候觉得无聊/焦虑"
→ 方案：骨架屏+个性化加载动画+等待时推送今日萌宠冷知识
→ 成本：2周开发 vs 3个月重构，用户感知等待时间减少60%
```
